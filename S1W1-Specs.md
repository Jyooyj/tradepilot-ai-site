# S1W1 Specs | TradePilot AI 第一阶段规格说明

## 项目名称

TradePilot AI | 拿货搭子：进货选品与爆款测款智能体

## 目标用户

- 大学生创业者。
- 校园摊位、社群团购和小微零售经营者。
- 义乌小商品、饰品、文创、家居等品类的新手拿货用户。
- 小红书、抖音、视频号等内容电商卖家。
- 需要低成本试单、快速测款和复盘的初创团队。

## 核心痛点

- 凭感觉拿货，缺少结构化判断依据。
- 成本、售价、MOQ、物流和售后风险难以统一评估。
- 不知道商品是否具备内容传播和爆款潜力。
- 多个候选商品之间缺少可比较的优先级。
- 进货后缺少产品库沉淀和测款复盘机制。
- 图片识别或 AI 接口不稳定时，演示和业务流程容易中断。

## 产品目标

TradePilot AI 的目标是把一次小商品进货决策拆解为可执行、可复盘、可比较的流程：

```text
商品识别 -> 进货信息采集 -> 利润测算 -> 风险判断 -> 内容测款建议 -> 产品库沉淀 -> 候选产品 PK -> 测款复盘
```

第一阶段重点验证：用户是否能在一个可运行原型中完成从商品输入到进货报告，再到保存、对比和复盘的完整闭环。

## 核心功能范围

### 商品输入

- 支持上传商品图片。
- 支持图片预览。
- 支持 AI 图片识别。
- 支持识别失败后的手动填写兜底。

### 进货信息采集

- 商品名称、品类、材质、目标人群。
- 成本价、预计售价、MOQ。
- 供应商、渠道、竞品价格。
- 物流、售后和其他风险说明。

### 进货决策报告

- 利润空间和毛利率测算。
- 首批压货成本估算。
- 风险等级和风险原因。
- 爆款潜力评分。
- 内容测款标题、卖点和测试建议。
- 是否建议小批量试单。

### 产品库

- 保存历史进货判断记录。
- 查看和恢复历史报告。
- 支持本地备份导出和导入。

### 候选产品 PK

- 对多个候选商品进行评分、利润、风险和内容潜力对比。
- 给出优先拿样或试单建议。

### 测款复盘

- 根据测试反馈生成下一步建议。
- 支持判断补货、优化内容、换图、调价或停止进货。

### 报告导出

- 支持下载文本报告。
- 方便评审、复盘和团队沟通。

## 用户流程

1. 用户进入首页。
2. 选择评委快速演示、游客模式或真实图片识别路径。
3. 上传商品图片，或选择内置示例商品。
4. 系统识别商品信息，或用户手动填写商品信息。
5. 用户补充成本、售价、MOQ、渠道、物流等进货变量。
6. 系统生成进货决策报告。
7. 用户保存报告到产品库。
8. 用户对多个候选商品进行 PK。
9. 用户进入测款复盘，记录测试反馈并获得下一步建议。
10. 用户下载报告或导出产品库备份。

## MVP 验收标准

- 可以本地运行并完成生产构建。
- 可以打开项目首页并进入核心功能。
- 可以通过示例商品稳定完成评委快速演示。
- 可以上传图片并尝试调用识别接口。
- 图片识别失败时，可以手动填写信息并继续生成报告。
- 生成的报告包含利润、风险、评分和内容测款建议。
- 报告可以保存到产品库。
- 产品库记录可以被查看、恢复和导出。
- 候选产品 PK 可以比较多个保存记录。
- 测款复盘可以输出下一步经营建议。
- 项目不包含 API Key、`.env`、`node_modules` 或 `dist` 提交内容。

## 非目标范围

- 第一阶段不追求完整商业化账号体系。
- 第一阶段不强制实现跨设备云端同步。
- 第一阶段不覆盖所有商品品类。
- 第一阶段不直接替代真实市场调研和供应链审核。
- 第一阶段不承诺 AI 图片识别在所有图片上都完全准确。
- 第一阶段不处理真实支付、订单、库存和物流系统。

## 后续扩展方向

- 接入 Supabase Auth，实现登录、游客模式升级和跨设备同步。
- 将产品库迁移到云端数据库。
- 增加更多商品品类模板和评分规则。
- 引入真实测款数据，如浏览、收藏、询单、成交和复购。
- 支持批量导入候选商品。
- 支持更完整的报告导出格式。
- 增加团队协作和多角色评审能力。
