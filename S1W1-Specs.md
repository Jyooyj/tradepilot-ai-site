# S1W2 Skills｜TradePilot AI 关键技能与工作流说明

## 项目名称

TradePilot AI｜进货决策与爆款测款智能体

## 复赛目标

复赛阶段重点验证项目的技术可行性和核心业务闭环。TradePilot AI 提交的核心能力不是单个页面功能，而是一组围绕“小商品进货决策”的 AI Skills 与可运行 Prototype：

```text
商品识别 → 信息采集 → 利润测算 → 风险判断 → 爆款评分 → 内容测款 → 产品库沉淀 → 候选PK → 测款复盘
```

---

## Skill 1：商品视觉识别 Skill

### 输入

- 用户上传的商品图片
- 可选补充线索，如“饰品、文创、家居、小商品”等

### 处理流程

系统通过 Vercel Serverless Function 调用阿里云百炼视觉模型，对商品图片进行识别，并要求模型返回结构化 JSON。识别重点包括商品名称、品类、材质、适用人群、销售场景、关键词和视觉判断原因。

### 输出

- 商品名称
- 商品品类
- 材质 / 风格
- 适用人群
- 预估售价区间
- 关键词
- 视觉识别置信度

### 对应页面 / 按钮

- 页面：开始判断
- 按钮：上传图片、AI识别商品

### 评委验证方式

上传一张商品图片，点击“AI识别商品”，检查右侧表单是否自动填入商品信息。

---

## Skill 2：进货信息采集 Skill

### 输入

- 成本价
- 预计售价
- MOQ 起订量
- 供应商 / 渠道
- 竞品价格
- 物流与售后风险
- 目标人群

### 处理流程

系统将影响进货判断的关键变量进行结构化采集，使后续利润测算、风险分析和爆款评分不依赖模糊描述，而是基于可比较的数据字段。

### 输出

- 结构化进货信息表单
- 可用于报告生成的产品对象

### 对应页面 / 按钮

- 页面：开始判断
- 区域：右侧进货信息表单

### 评委验证方式

修改成本价、售价、MOQ 等字段，观察报告中的利润分析和进货建议是否随输入变化。

---

## Skill 3：利润与风险分析 Skill

### 输入

- 商品基础信息
- 成本价
- 预计售价
- MOQ
- 竞品价格
- 渠道、供应商和物流信息

### 处理流程

系统根据用户填写的信息计算利润空间，并从多个角度判断进货风险，包括压货风险、同质化风险、价格风险、供应链风险、物流风险和信息不完整风险。

### 输出

- 利润空间判断
- 风险等级
- 风险原因说明
- 是否建议小批量试单

### 对应页面 / 按钮

- 页面：进货报告
- 按钮：生成进货决策报告

### 评委验证方式

填写不同的成本价和售价，生成报告，查看系统是否给出不同的利润判断和风险提示。

---

## Skill 4：爆款潜力评分 Skill

### 输入

- 商品识别结果
- 利润信息
- 目标人群
- 内容传播关键词
- 供应链与拿货信息

### 处理流程

系统从利润空间、视觉吸引力、内容传播性、目标人群匹配度、供应链稳定性、试单风险等维度进行评分，并生成综合爆款潜力分数。

### 输出

- 综合评分
- 分项评分
- 评分依据
- 进货优先级

### 对应页面 / 按钮

- 页面：进货报告
- 区域：综合评分、分项评分卡片

### 评委验证方式

生成报告后，查看是否出现总分、分项评分和对应解释。

---

## Skill 5：内容测款方案生成 Skill

### 输入

- 商品名称
- 商品卖点
- 目标人群
- 价格区间
- 使用场景
- 平台方向，如小红书 / 抖音

### 处理流程

系统基于商品信息生成适合内容平台测款的表达方向，包括标题、卖点、种草角度、短视频/图文内容切入点和测试建议。

### 输出

- 小红书 / 抖音测款标题
- 内容卖点
- 种草角度
- 测款建议

### 对应页面 / 按钮

- 页面：进货报告
- 区域：内容测款建议

### 评委验证方式

查看报告中是否给出可直接用于内容平台测试的文案方向和测款建议。

---

## Skill 6：候选产品 PK Skill

### 输入

- 已保存的多个产品记录
- 每个产品的评分、风险、售价、成本和建议

### 处理流程

系统读取“我的产品库”中的历史产品记录，对多个候选产品进行横向比较，帮助用户判断哪一个更适合作为优先拿样或重点测款对象。

### 输出

- 候选产品列表
- 最高优先级商品
- 推荐原因
- 横向对比结果

### 对应页面 / 按钮

- 页面：候选产品PK
- 前置动作：保存至少一条进货报告到产品库

### 评委验证方式

保存多条报告后进入“候选产品PK”，检查是否能看到产品比较和优先级判断。

---

## Skill 7：测款复盘 Skill

### 输入

- 商品历史记录
- 测款数据
- 销量、互动、点击、收藏、转化等反馈

### 处理流程

系统根据产品记录和测款表现，判断是否应该继续补货、优化内容、调整定价、换图测试或停止进货。

### 输出

- 复盘结论
- 下一步建议
- 补货 / 优化 / 停止判断

### 对应页面 / 按钮

- 页面：测款复盘
- 按钮：保存本次复盘到产品库

### 评委验证方式

进入“测款复盘”页面，填写或查看测款数据，检查系统是否输出复盘建议。

## 总体评测路径

```text
1. 打开项目首页
2. 点击“评委快速演示”查看内置案例
3. 点击“开始进货判断”
4. 上传商品图片并点击 AI 识别
5. 补充进货信息
6. 生成进货决策报告
7. 保存到我的产品库
8. 查看候选产品PK
9. 查看测款复盘
10. 下载报告
```

## Skills 与 Prototype 对应关系

| Skill | 对应模块 | 可验证结果 |
|---|---|---|
| 商品视觉识别 | 上传图片 / AI识别商品 | 输出结构化商品信息 |
| 进货信息采集 | 右侧表单 | 形成完整进货变量 |
| 利润与风险分析 | 进货报告 | 输出风险和建议 |
| 爆款潜力评分 | 评分卡片 | 输出总分和分项解释 |
| 内容测款方案 | 内容建议模块 | 输出标题和种草角度 |
| 候选产品 PK | PK 页面 | 比较多个历史产品 |
| 测款复盘 | 复盘页面 | 输出下一步经营建议 |
