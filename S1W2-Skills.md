# S1W2 Skills｜TradePilot AI 关键技能与工作流说明

## 1. 项目名称

TradePilot AI｜拿货搭子：进货选品与爆款测款智能体

## 2. 阶段目标

S1W2 阶段的目标是在 S1W1 Specs 的基础上，跑通一个可体验、可演示、可验证的 Prototype。

本阶段重点不是只做一个展示页面，而是实现一组围绕“小商品进货决策”的 AI Skills，并将这些 Skills 串联成完整工作流：

商品识别 → 信息采集 → 利润测算 → 风险判断 → 爆款评分 → 内容测款 → 产品库沉淀 → 候选 PK → 测款复盘

项目希望通过这些 Skills，帮助用户在进货前减少主观判断，提高选品、测款和复盘效率。

---

## 3. 核心工作流

TradePilot AI 的核心工作流如下：

1. 用户进入游客演示模式
2. 上传产品图片或手动填写产品信息
3. 系统识别或整理产品基础信息
4. 用户补充拿货价、建议售价、MOQ、材质、渠道、供应商等信息
5. 系统生成进货决策报告
6. 用户将产品保存到“我的产品库”
7. 系统支持产品库搜索、筛选和排序
8. 用户可对候选产品进行 PK
9. 用户输入测款数据进行复盘
10. 系统输出是否继续测款、补货或放弃的建议
11. 用户可下载 HTML 可视化报告
12. 用户可通过反馈入口提交建议

---

## 4. Skill 1：商品视觉识别 Skill

### 4.1 功能说明

商品视觉识别 Skill 用于辅助用户从商品图片中提取基础产品信息。用户上传产品图片后，系统调用视觉模型识别商品的品类、材质、风格、适用人群和内容关键词，并辅助回填到表单中。

### 4.2 输入

- 用户上传的商品图片
- 可选的产品补充描述

### 4.3 处理流程

1. 用户选择商品图片。
2. 前端对图片进行预览。
3. 如果图片较大，前端会尝试进行压缩。
4. 系统调用图片识别接口。
5. 视觉模型返回结构化商品信息。
6. 前端将识别结果回填到产品信息表单中。
7. 用户可以继续手动修改或补充信息。

### 4.4 输出

- 产品名称
- 产品类型
- 材质 / 风格
- 目标人群
- 销售渠道建议
- 内容关键词
- 适合的使用场景

### 4.5 价值

该 Skill 降低了用户填写产品信息的门槛，尤其适合进货场景中“看到商品但不知道如何描述和归类”的情况。

## 5. Skill 2：产品身份识别与品类校验 Skill

### 5.1 功能说明

产品身份识别与品类校验 Skill 用于解决不同小商品之间容易串模板的问题。例如，钥匙扣、文创挂饰、发圈、耳夹、手链等商品在外观上可能存在相似元素，如果仅依靠单一关键词，容易导致报告内容错误。

### 5.2 输入

- 产品名称
- 产品类型
- 材质
- 关键词
- 目标人群
- 渠道
- 场景
- 用户备注

### 5.3 处理逻辑

系统综合多个字段进行证据判断，而不是只依赖单个关键词。不同产品会被识别为更细的身份类型，例如：

- 发饰类
- 耳夹 / 耳饰类
- 手链 / 手绳类
- 文创挂饰 / 钥匙扣类
- 文创纸品类
- 手机配件类
- 家居生活类
- 日用小商品类

### 5.4 输出

- 产品身份标签
- 品类方向
- 可使用词汇
- 禁用词汇
- 内容生成约束

### 5.5 价值

该 Skill 可以减少报告中出现跨品类串词的问题，例如：

- 文创挂饰报告中不应出现“发圈”“扎发”“低丸子头”
- 发圈报告中不应出现“钥匙扣”“果核”“包包挂件”
- 耳夹报告中不应出现“手链”“扎发”
- 真实手链不应被强制改成钥匙扣或挂饰

## 6. Skill 3：进货信息采集 Skill

### 6.1 功能说明

进货信息采集 Skill 用于收集进货判断所需的基础字段，帮助用户将模糊的选品想法转化为可分析的数据。

### 6.2 输入字段

- 产品名称
- 产品类型
- 拿货价
- 建议售价
- MOQ 最小起订量
- 材质
- 目标人群
- 销售渠道
- 供应商信息
- 竞品价格
- 物流风险
- 备注

### 6.3 处理方式

系统将用户输入的信息统一整理为结构化产品对象，并用于后续利润测算、风险判断、内容测款和产品库保存。

### 6.4 输出

- 结构化产品信息
- 用于报告生成的产品数据
- 用于产品库保存的记录对象

### 6.5 价值

该 Skill 帮助用户把“凭感觉选品”转化为“基于字段和指标的进货判断”。

---

## 7. Skill 4：利润测算 Skill

### 7.1 功能说明

利润测算 Skill 用于帮助用户判断产品是否具备基本盈利空间。

### 7.2 输入

- 拿货价
- 建议售价
- MOQ
- 竞品价格
- 物流和包装风险信息

### 7.3 计算内容

- 单件成本
- 单件利润
- 毛利率
- 首批压货资金
- 售价合理性
- 价格带风险

### 7.4 输出

- 单件利润
- 毛利率
- 首批投入金额
- 价格带判断
- 是否需要补充成本或售价信息

### 7.5 价值

该 Skill 可以避免用户只看产品外观而忽视成本、售价和 MOQ 带来的压货风险。

---

## 8. Skill 5：进货风险判断 Skill

### 8.1 功能说明

进货风险判断 Skill 用于识别产品在供应、物流、库存、同质化和内容转化方面的潜在风险。

### 8.2 判断维度

- MOQ 是否过高
- 首批压货资金是否过大
- 材质是否容易带来质量争议
- 商品是否易碎、易变形或包装成本高
- 供应商是否支持混批、补货和定制
- 产品是否存在严重同质化
- 产品卖点是否足够明确

### 8.3 输出

- 风险等级
- 风险说明
- 规避建议
- 是否建议先拿样
- 是否建议小批量测试

### 8.4 价值

该 Skill 帮助新手卖家在进货前提前发现风险，减少盲目囤货。

---

## 9. Skill 6：爆款潜力评分 Skill

### 9.1 功能说明

爆款潜力评分 Skill 用于综合判断一个产品是否值得优先测款。

### 9.2 评分维度

- 利润与价格带
- 内容潜力
- 渠道适配
- 供应稳定
- 风险可控
- 信息完整度

### 9.3 输出

- 综合评分
- 各维度评分
- AI 建议
- 是否适合拿样
- 是否适合内容测款
- 是否需要补充信息

### 9.4 价值

该 Skill 将原本主观的选品判断转化为相对结构化的评分结果，方便用户快速比较不同产品。

---

## 10. Skill 7：内容测款建议 Skill

### 10.1 功能说明

内容测款建议 Skill 用于帮助用户将产品转化为适合小红书、抖音等平台测试的内容方案。

### 10.2 输入

- 产品名称
- 产品类型
- 材质
- 目标人群
- 渠道
- 产品卖点
- 使用场景

### 10.3 输出

- 小红书标题建议
- 小红书封面建议
- 图文结构建议
- 抖音短视频脚本
- 推荐话题标签
- 跨平台搜索关键词
- 用户痛点切入角度
- 产品差异化表达

### 10.4 价值

该 Skill 帮助用户从“我觉得这个产品不错”进一步走向“我应该如何测试这个产品有没有市场”。

---

## 11. Skill 8：供应商沟通清单 Skill

### 11.1 功能说明

供应商沟通清单 Skill 用于帮助用户在进货前明确需要向供应商确认的问题。

### 11.2 输出内容

- 是否支持混批
- 是否支持补货
- 是否支持定制包装
- 起订量是否可谈
- 是否可以提供实拍图
- 是否可以寄样
- 是否有质量问题售后
- 是否存在色差、瑕疵、易损风险
- 发货周期和物流方式

### 11.3 价值

该 Skill 可以降低新手与供应商沟通时的信息不对称，减少后续退换货和压货风险。

---

## 12. Skill 9：我的产品库 Skill

### 12.1 功能说明

我的产品库 Skill 用于保存用户已经分析过的产品，形成长期选品记录。

### 12.2 支持能力

- 保存产品
- 查看产品详情
- 搜索产品
- 按品类筛选
- 按评分排序
- 按时间排序

### 12.3 数据存储

游客模式下使用 localStorage 存储，key 为：

tradepilot_local_records

### 12.4 价值

该 Skill 让用户的每一次选品判断都被记录下来，方便后续复盘和横向对比。

---

## 13. Skill 10：候选产品 PK Skill

### 13.1 功能说明

候选产品 PK Skill 用于对产品库中的多个产品进行横向比较，帮助用户判断优先测试哪个产品。

### 13.2 对比维度

- 综合评分
- 单件利润
- 毛利率
- 内容潜力
- 渠道适配
- 风险等级
- 供应稳定性

### 13.3 输出

- 推荐优先测试产品
- 推荐理由
- 产品对比说明
- 需要谨慎测试的产品提示

### 13.4 价值

该 Skill 适合用户面对多个候选商品时做排序和取舍，避免同时进太多货导致预算分散。

---

## 14. Skill 11：测款复盘 Skill

### 14.1 功能说明

测款复盘 Skill 用于根据用户输入的测款数据，判断产品后续是否值得继续投入。

### 14.2 输入字段

- 浏览量
- 点赞数
- 收藏数
- 评论数
- 询单数
- 成交数
- 测款成本

### 14.3 计算指标

- 互动率
- 询单率
- 转化率
- 单均测款成本

### 14.4 输出

- 测款表现判断
- 是否建议继续测款
- 是否建议补货
- 是否建议优化内容后再测
- 是否建议放弃该产品

### 14.5 价值

该 Skill 将测款结果从“凭感觉复盘”转化为“基于数据复盘”，帮助用户形成持续优化能力。

### 15.3 价值

该 Skill 方便用户进行团队讨论、比赛展示、项目沉淀和长期复盘。
