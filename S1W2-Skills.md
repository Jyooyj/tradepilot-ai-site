# S1W2 Skills | TradePilot AI 关键技能与工作流说明

## 项目名称

TradePilot AI | 拿货搭子：进货选品与爆款测款智能体

## 工作流总览

TradePilot AI 的核心能力不是单点页面功能，而是一组围绕“小商品进货决策”的 AI Skills 与可运行 Prototype：

```text
商品视觉识别 -> 进货信息采集 -> 利润测算与风险判断 -> 内容测款建议 -> 产品库沉淀 -> 候选产品 PK -> 测款复盘
```

## Skill 1：商品视觉识别 Skill

### 输入

- 用户上传的商品图片。
- 可选补充线索，如饰品、文创、家居、小商品等。

### 处理流程

系统通过 Vercel Serverless Function 调用图片识别能力，对商品图片进行识别，并尽量返回结构化结果。识别重点包括商品名称、品类、材质、风格、适用人群、销售场景、关键词和视觉判断依据。

### 输出

- 商品名称。
- 商品品类。
- 材质和风格。
- 目标人群。
- 销售场景。
- 关键词。
- 识别置信度或识别说明。

### 评委验证方式

上传一张商品图片，点击 AI 识别商品，检查表单是否自动补全商品信息。若接口异常，可验证手动填写兜底流程是否仍能继续生成报告。

## Skill 2：进货信息采集 Skill

### 输入

- 商品基础信息。
- 成本价。
- 预计售价。
- MOQ 起订量。
- 供应商和渠道。
- 竞品价格。
- 物流和售后风险。
- 目标人群和内容关键词。

### 处理流程

系统将影响进货判断的关键变量结构化，形成可用于利润测算、风险判断和内容测款建议的数据对象，避免只依赖模糊描述。

### 输出

- 完整的商品进货信息表单。
- 可用于报告生成的产品对象。
- 可保存到产品库的候选商品记录。

### 评委验证方式

修改成本价、预计售价、MOQ、渠道和风险描述，观察生成报告中的利润、风险和建议是否随输入变化。

## Skill 3：利润测算与风险判断 Skill

### 输入

- 商品基础信息。
- 成本价和预计售价。
- MOQ。
- 竞品价格。
- 渠道、供应商、物流和售后信息。

### 处理流程

系统根据用户填写的信息计算毛利、毛利率和首批压货成本，并从库存、价格、同质化、供应链、物流和信息完整度等维度判断风险。

### 输出

- 利润空间判断。
- 首批压货成本估算。
- 风险等级。
- 风险原因说明。
- 是否建议小批量试单。
- 进货优先级建议。

### 评委验证方式

填写不同成本、售价和 MOQ，生成报告后检查利润测算、风险提示和进货建议是否发生合理变化。

## Skill 4：内容测款建议 Skill

### 输入

- 商品名称。
- 商品卖点。
- 目标人群。
- 价格区间。
- 使用场景。
- 内容平台方向，如小红书、抖音或校园社群。

### 处理流程

系统基于商品信息生成适合内容平台测试的表达方向，包括标题、卖点、种草角度、短视频或图文切入点，以及可执行的测款动作。

### 输出

- 内容测款标题。
- 核心卖点。
- 种草角度。
- 图文或短视频建议。
- 测款动作和观察指标。

### 评委验证方式

查看报告中是否给出可直接用于内容平台测试的标题、卖点、脚本方向和测款建议。

## Skill 5：产品库沉淀 Skill

### 输入

- 已生成的进货决策报告。
- 商品基础信息。
- 评分、风险、利润和内容建议。
- 用户在测款复盘中的补充反馈。

### 处理流程

系统将每次进货判断保存为可回看的产品记录。游客模式下记录保存在浏览器 localStorage 中，并支持导出和导入备份。

### 输出

- 我的产品库记录。
- 可恢复查看的历史报告。
- 产品库备份文件。
- 支持后续 PK 和复盘的数据基础。

### 评委验证方式

生成报告后点击保存到产品库，进入产品库检查记录是否存在，并尝试查看、恢复、导出或导入备份。

## Skill 6：候选产品 PK Skill

### 输入

- 产品库中的多个候选商品。
- 每个商品的评分、利润、风险、售价、成本、渠道适配和内容潜力。

### 处理流程

系统读取产品库记录，对多个候选商品进行横向比较，帮助用户判断哪个商品更适合作为优先拿样、试单或持续观察对象。

### 输出

- 候选产品列表。
- 横向对比结果。
- 推荐优先级。
- 推荐原因。
- 可回看单个商品报告的入口。

### 评委验证方式

保存至少两个商品后进入候选产品 PK 页面，检查是否能看到产品对比和优先级判断。

## Skill 7：测款复盘 Skill

### 输入

- 商品历史记录。
- 测款反馈。
- 浏览、互动、收藏、询单、成交等表现数据。
- 用户对价格、内容和渠道的补充说明。

### 处理流程

系统根据产品记录和测款表现，判断下一步应该补货、优化内容、换图、调价、继续观察或停止进货。

### 输出

- 复盘结论。
- 下一步经营建议。
- 补货 / 优化 / 停止判断。
- 可保存回产品库的复盘记录。

### 评委验证方式

进入测款复盘页面，填写或查看测款数据，检查系统是否输出清晰的下一步建议，并支持保存复盘结果。

## Skills 与 Prototype 对应关系

| Skill | 对应模块 | 可验证结果 |
|---|---|---|
| 商品视觉识别 | 上传图片 / AI 识别商品 | 输出结构化商品信息 |
| 进货信息采集 | 进货信息表单 | 形成完整进货变量 |
| 利润测算与风险判断 | 进货决策报告 | 输出利润、风险和建议 |
| 内容测款建议 | 内容建议模块 | 输出标题、卖点和测款动作 |
| 产品库沉淀 | 我的产品库 | 保存、查看、恢复和备份记录 |
| 候选产品 PK | 候选产品 PK 页面 | 比较多个历史商品 |
| 测款复盘 | 测款复盘页面 | 输出下一步经营建议 |

## 总体评测路径

1. 打开项目首页。
2. 进入评委快速演示或游客模式。
3. 选择示例商品，或上传图片并尝试 AI 识别。
4. 补充进货信息。
5. 生成进货决策报告。
6. 保存到我的产品库。
7. 查看候选产品 PK。
8. 查看测款复盘。
9. 下载报告或导出产品库备份。
